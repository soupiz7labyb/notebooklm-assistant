# Быстрый старт

## Шаг 1: Установка зависимостей

```bash
npm install
```

## Шаг 2: Сборка проекта

```bash
npm run dev
```

Это создаст папку `dist` с собранным расширением.

## Шаг 3: Настройка OAuth (упрощенный вариант)

Для начала можно использовать расширение без полной настройки OAuth. Расширение будет работать с cookies браузера, если вы уже залогинены в `notebooklm.google.com`.

Если хотите использовать Google OAuth:

1. Откройте [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте проект
3. Включите **People API**
4. Создайте **OAuth 2.0 Client ID** типа **Web application**
5. Скопируйте Client ID
6. Вставьте в `src/manifest.json` вместо `YOUR_CLIENT_ID_HERE`

**Примечание:** После первой загрузки расширения в Chrome, скопируйте Extension ID и добавьте redirect URI:
```
https://<ваш-extension-id>.chromiumapp.org/
```

## Шаг 4: Загрузка в Chrome

1. Откройте `chrome://extensions/`
2. Включите **"Режим разработчика"**
3. Нажмите **"Загрузить распакованное расширение"**
4. Выберите папку **`dist`** (она появится после `npm run dev`)

## Шаг 5: Первое использование

1. Убедитесь, что вы залогинены в `notebooklm.google.com`
2. Откройте расширение (кликните на иконку)
3. Нажмите "Sign in with Google" (если настроили OAuth) или просто используйте cookies
4. Выберите или создайте блокнот

## Важно: Настройка API эндпоинтов

После загрузки расширения:

1. Откройте `notebooklm.google.com` в Chrome
2. Откройте **DevTools > Network**
3. Создайте блокнот и добавьте источник
4. Найдите реальные API запросы
5. Обновите эндпоинты в `src/services/notebooklm-api.ts`
6. Пересоберите: `npm run dev`

## Решение проблем

### Папки `dist` нет
Выполните `npm run dev` или `npm run build` - папка создастся автоматически.

### Ошибка OAuth
Можно использовать расширение без OAuth, если вы залогинены в NotebookLM через браузер.

### Не работает загрузка блокнотов
Нужно обновить API эндпоинты в `src/services/notebooklm-api.ts` на основе реальных запросов из Network tab.
